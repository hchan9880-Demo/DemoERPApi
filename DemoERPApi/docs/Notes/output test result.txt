The Quickest Built-in Way (TRX to HTML/Excel)
Visual Studio logs all test executions into a standard standard .trx (Test Results XML) file. You can generate this via the CLI and instantly turn it into an interactive HTML or Excel report.

Open your terminal (or Visual Studio Developer PowerShell) at your root solution directory.

Run the following command to execute the tests and output a logger file:

Bash
dotnet test --logger "trx;LogFileName=test_results.trx"
Look in your test project's TestResults/ folder for test_results.trx.

To convert that .trx into a beautiful HTML/PDF doc:
Install the global liquid report generator tool:

Bash
dotnet tool install -g trx2html
trx2html TestResults/test_results.trx
This will produce a styled HTML page with charts summarizing your 136 tests, which you can open in any browser and choose Print -> Save as PDF.

LiquidTestReports (Markdown Dashboard)
The CLI tool package name on NuGet is actually LiquidTestReports.Cli. Since it's currently in beta, you just need to add the --prerelease flag.

Install the tool:

PowerShell
dotnet tool install --global LiquidTestReports.Cli --prerelease
Run your tests to generate a .trx file:

PowerShell
dotnet test --logger "trx;LogFileName=test_results.trx"
Convert the .trx directly to a beautiful Markdown report:

PowerShell
liquid --inputs "File=**/test_results.trx;Format=Trx" --output-file Milestone_4_7_Report.md
This spits out a clean Milestone_4_7_Report.md in your project folder, which you can open or print to PDF!

Step 3: Run the Script
Because Windows has safety restrictions on running scripts, you just need to call it with a dot-slash (.\) so PowerShell knows to look in your current folder:

In your terminal, type:

PowerShell

Press Enter.

Note: If you get an error saying “scripting is disabled on this system”, you just need to bypass the restriction for this one run by executing it like this instead:

PowerShell
PowerShell -ExecutionPolicy Bypass -File .\GenerateReport.ps1
